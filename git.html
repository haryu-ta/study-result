コマンド

		
■ 変更ファイルの取り消し(add前ファイルの取消)
git checkout HEAD -- [ファイル名]
→ファイル変更が消える

git checkout HEAD -- [ディレクトリ名]
→ディレクトリ単位で取り消し

git checkout -- .
→全変更の取り消し

■ addの取消（Unstageのみ）
git reset HEAD [ファイル名]
git reset HEAD [ディレクトリ名]
git reset HEAD .

→ファイル変更は残る

→直前のコミットに追加でコミット内容を追加
※pushした後に--amendすることは厳禁

■ commitの取消（直前のコミットのみ取消。addも取消）
git reset --mixed HEAD^

■ commitの取消（直前のコミットのみ取消。addされた状態）
git reset --soft HEAD^

■ commitの取消（直前のコミットのみ取消。ファイル変更もなし）
git reset --hard HEAD^
	
■ リモートから情報を取得
git fetch
git merge [リモート名]/[取込ブランチ]
→現在いるブランチに取込ブランチをMerge
				
git pull [リモート名] [ブランチ名]
→ fetchとmergeを1発で実施する

■ 履歴を整えた形で統合
git rebase [親ブランチ名]
→pushしたコミットをリベースは絶対にNG

■ commitのやり直し（追加コミット）
git commit --ammend
→直前の１回の取り消し

git rebase -i HEAD~5
→直前５回の取り消し（５回の中でどれとどれみたいな選択可能）

■ 作業を一時退避
git stash

■ 退避データを確認
git stash list

■ 退避データを復元

# 直前の作業を復元
git stash apply
git stash apply --index
→ indexオプションを付加するとステージング状態（git addしたもの）のものも復元

# 特定の作業を復元
git stash apply [スタッシュ名]
git stash apply stash@{1}

■ ローカルのリモートURL変更
git remote set-url origin [URL]

■ 親リポジトリのマージ
cd [マージしたいリポジトリ]
git remote add parent [親のリポジトリURL]
git remote -v
git checkout master
git checkout -b merge_branch
git fetch parent
git merge parent/[マージ元ブランチ名]
git push origin merge_branch

→ 現在作業中のブランチにMergeする。
